# Legal and Safety Checklist

This checklist keeps the skill analysis-focused and non-infringing.

## Disallowed by default

- [ ] Do not explain how to decompile, extract, rip, dump, scrape, or bypass protections.
- [ ] Do not reproduce private scripts or code from a game.
- [ ] Do not copy proprietary maps, thumbnails, UI, icons, branding, characters, sounds, animations, or models.
- [ ] Do not provide exact clone instructions.
- [ ] Do not create a similar game concept unless the user explicitly asks after research.
- [ ] Do not create implementation code unless explicitly requested and the code is original.

## Allowed

- [ ] Analyze visible gameplay behavior.
- [ ] Analyze public store pages and official materials.
- [ ] Analyze screenshots and footage.
- [ ] Analyze scripts/files the user owns or has permission to inspect.
- [ ] Infer likely architecture cautiously.
- [ ] Extract high-level design lessons.
- [ ] Compare patterns across games.
- [ ] Suggest lawful, original alternatives only after explicit creation request.

## Required wording

Use:

- “Observed” for visible facts.
- “Confirmed” for reliable-source-supported facts.
- “Likely” or “appears to” for strong inferences.
- “Possible” for weak inferences.
- “Unknown from available evidence” when evidence is missing.

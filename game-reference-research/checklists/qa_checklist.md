# QA Checklist

Use this before delivering any report.

## Evidence

- [ ] Every major factual claim has evidence or is marked as an inference.
- [ ] Official/user-provided sources are prioritized over community sources.
- [ ] Source type is listed for each evidence item.
- [ ] Confidence is assigned to important claims.
- [ ] Unknowns and limitations are listed.

## Analysis quality

- [ ] Core loop is explained at short-, medium-, and long-term levels.
- [ ] Player verbs are named clearly.
- [ ] Success, failure, reward, and retry are explained.
- [ ] UX/onboarding is analyzed, not just described.
- [ ] Systems are decomposed into purpose, evidence, likely data, and confidence.
- [ ] Visual style is described with concrete traits.
- [ ] Game feel discusses timing, responsiveness, feedback, camera, and polish.
- [ ] Level/map design covers guidance, progression, landmarks, and flow.

## Technical accuracy

- [ ] Architecture claims are cautious when scripts are not provided.
- [ ] Roblox-specific assumptions use “likely/appears/possible” wording.
- [ ] Client/server/shared classification is not overstated.
- [ ] Security/performance risks are tied to visible behavior or supplied code.

## Safety and ethics

- [ ] No decompilation or ripping advice.
- [ ] No cloned code/assets/UI/maps/thumbnails/branding.
- [ ] No step-by-step clone build guide unless the user explicitly asked for an original lawful plan.
- [ ] Monetization analysis does not become a monetization plan unless requested.
- [ ] Lessons are high-level and non-infringing.

## Output

- [ ] Tables are readable.
- [ ] Final takeaway is concise.
- [ ] The report answers the user's actual focus.

# Workflow

## 1. Intake

Collect the user's input:

- Game link
- Platform
- Screenshots/footage
- Specific focus
- Any scripts/files they own
- Desired output depth

## 2. Source map

Create a short source plan:

- User-provided evidence
- Official pages
- Store pages
- Structured databases
- Professional/academic sources
- Curated UI/style archives
- Community sources only if needed

## 3. Evidence extraction

For each evidence item:

- Record source type
- Record what claim it supports
- Mark confidence
- Capture limitations

## 4. Analysis

Analyze in this order:

1. Identity and genre
2. Core loop
3. Player experience/onboarding
4. UX/UI
5. Main systems
6. Visual style
7. Game feel
8. Level/map design
9. Technical architecture
10. Monetization if visible/requested
11. Accessibility/usability
12. Lessons and unknowns

## 5. QA

Run the QA checklist:

- Evidence separation
- Confidence labels
- Safety/IP review
- Unknowns
- No accidental clone plan

## 6. Final output

Deliver a report with:

- Clear tables
- Evidence log
- Careful wording
- High-level lessons
- Unknowns
- Final takeaway

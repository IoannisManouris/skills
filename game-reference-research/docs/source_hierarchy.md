# Source Hierarchy

Use this hierarchy to decide what evidence to trust.

## Tier 0: User-provided evidence

Examples:

- Screenshots
- Gameplay footage
- Scripts/files the user owns
- Analytics screenshots
- Direct project notes

Best for: exact details the user wants analyzed.

Caveat: verify whether scripts/assets are owned or allowed to be inspected if unclear.

## Tier 1: Direct game evidence

Examples:

- Playing the game
- Captured gameplay
- Official trailer footage
- Official screenshots
- Store-page media

Best for: visible systems, UI, gameplay loop, style, game feel, onboarding.

Caveat: footage may be outdated or edited.

## Tier 2: Official sources

Examples:

- Developer site
- Publisher site
- Roblox page
- Steam page
- App Store / Google Play page
- Patch notes
- Credits
- Manuals
- Docs

Best for: current facts, descriptions, credits, platform availability, official positioning.

Caveat: marketing copy can exaggerate or omit weaknesses.

## Tier 3: Structured databases

Examples:

- IGDB
- MobyGames
- Steam tags
- Platform metadata

Best for: discovery, release data, genre tags, companies, credits, screenshots, metadata.

Caveat: verify important facts against official sources.

## Tier 4: Professional and academic sources

Examples:

- GDC talks
- IGDA resources
- DiGRA papers
- HCI/UX papers
- Accessibility guidelines

Best for: frameworks, terminology, professional reasoning, historical context.

Caveat: may not describe the exact target game.

## Tier 5: Curated archives

Examples:

- Interface In Game
- Visual reference libraries
- Asset libraries with clear licenses

Best for: UI/style comparisons and pattern discovery.

Caveat: verify ownership and license before reuse.

## Tier 6: Community commentary

Examples:

- Reddit
- YouTube comments
- Fan wikis
- Forums
- Reviews

Best for: player perception, bug reports, sentiment clues, discovery.

Caveat: low confidence unless corroborated.
